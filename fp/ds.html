1. Introduction

Data is a vital component of any computer system. How we organize, store, and manage this data has an impact on the applications' usability, security and efficiency.

A data structure provides us with structured means of organizing data to aid in the actual processing and use of data.

At the same time, privacy is an important topic in the computing environment. Privacy ensures that sensitive information we manage in our systems is not accessed or used unexpectedly or illegitimately. When we consider data structures as a means of protecting privacy we can use them to develop systems that are efficient to use while being assured that the data is secure and will behave in a publishable manner.

2. What is a Data Structure?

Definition: A data structure is an organized way of storing, organizing and managing data in a computer that optimizes efficiency.

Objectives of Data Structures:

  1. Efficient Data Management - The goal of a data structure is to help organize data to ease the process of storing, possibly manipulating and retrieving that data.
  2. Memory Efficiency - Some data structures will reduce the amount of wasted memory from how the data organized, which will also result in performance gains.
  3. Support Algorithms - Examples exist where an algorithm requires a specific data structure to perform effectively.

Examples of Common Data Structures

Linear Structures: Array, Linked List, Stack, Queue

Non-linear Structures: Tree, Graph, Hash table

Importance

Without the use of a proper data structure, many applications will struggle to operate efficiently. In large use cases (e.g., a user in a database with millions of other entries in it) it is nearly impossible to locate the needed information if the data is not organized in a data structure.

3. The Role of Data Structures in Privacy

Online Safety

Online safety means protecting sensitive information (password, financial information, personal information) that can fall into abusive hands.

Data Structures are Critical enablers

  • Data is organized so security can restrict access.
  • Data structure can act as an enabler for secure tasks, such as encrypting sensitive data and validating data integrity.
  • Data structures can lock out the chance for access and to tamper or abuse consequently resulting in privacy violation.

Example

A bank uses a hash table to store passwords as hashed values. Regardless of the database being extracted by an unauthorized individual, unless they have a way to decrypt the hashed password bundled with additional knowledge of a tactic (social engineering), they will not know the password at all.

4. Tree Data Structure

Definition:

A tree is a data structure that consists of a root node, followed by connected edges, creating overall a hierarchy of node relationships.

Trees have following characteristics:

  • The top node is referred to as the root.
  • The nodes beneath the root are known as child nodes and the nodes above are known as parent nodes.
  • Each parent node can have multiple child nodes than the query, however trees can collectively have multiple hierarchy levels. The exception is binary trees, which each parent node has maximum of two child nodes joined as edges.

How trees enable privacy:

  1. Access Control Trees: Access privileges are assigned to a parent node, which is inherited by child nodes. For example, a private folder as a parent node would inherit private folders and files that child nodes can have.
  2. Merkle Trees: Each node contains a hash of its child nodes. If the final leaf node of a Merkle tree is changed, it would change the hash at the root node immediately identifying that tampering detected.

Outcomes

  • File Systems - an example of programs where trees organize folders and files in a hierarchy.
  • Blockchain - verifies that transactional integrity has not been compromised through a data structure.
  • Version Control Systems – Keeping track of changes across many files.

5. Graph Data Structure

Definition:

A graph is a structure of vertices (or nodes) and edges (or links) between vertexes/nodes.

Characteristics of Graphs:

  • There can be directed edges (you have a direction from one vertex/node to another vertex/node) or there can be undirected edges (edges do not have a direction).
  • Edges may or may not have weights, which could indicate cost, distance or some capacity.
  • Graphs model relationship(s) between some things.

How graphs can secure data:

  • Network Modeling: The graph models users, devices and their access to resources.
  • Anomaly Detection: You can study the graph for some anomaly or unusual behavior (like a user accessing inappropriate or unauthorized resources).
  • Access Control Design: Graphs use credentials from the subject/user to control who connects to what.

Applications for Graphs:

  • Social connections through social media (friend networks)
  • Network security (unidirectional or bidirectional data flow) or by monitoring data flow.
  • Access control of corporate networks.

6. Hash Table Data Structure

Definition: A hash table is a key-value data structure that uses a hash function to store data in a key-value pair.

Characteristics of Hash Tables:

  • Gives access to data fast using a key.
  • Values can be stored as hashes not as plain text.
  • Hashes can categorize items or keys that hash to the same value (collision) by an approach called chaining.

How hash tables can secure data:

  1. Passwords stored securely: original passwords will not be stored as hashes by unauthorized users to read.
  2. Checking data integrity - Hashes and checksums indicate data has not been tampered with.
  3. Authentication is fast.

Applications

  • Logins (where passwords are stored in a hash table)
  • Digital signatures
  • Web app session management

8. Data Structures Preventing Compromise

  1. Trees -If one byte in a tree is modified, the parent/root hash changes leading to a security compromise. We can also limit access permissions to only authorized browsed.
  2. Graphs -Relationship mapping between nodes (particular users, cli operations) allows user behviour to be examined too identify unusual behaviour, abnomral connections and patterns. We can thus protect against intrusion attacks, and loss of privilege attacks, as well as leaks from unauthorised access.
  3. Hash Tables -All sensitive data is pushed in as hashes so the data itself is never fully revealed. As hashes are very fast - we can quickly detect if any of the data has been manipulated / tampered. URL's appear as links of hid => our sensitive information is hidden and secure.

9. Conclusion

Data Structures = Efficient data organization

Tree = Hierarchy + integrity check

Graph = Relationship mapping + anomaly detection

Hash Table = Key-Value storage + confidentiality

Combined Roles:

Trees, graphs, and hash tables provide privacy and secure organization of data, as well as further prevention against breaches and exploitation of data. Their combined role represents the data structure methods of modern secure computing systems.
